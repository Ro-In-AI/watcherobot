# BLE 遥控机器人架构设计

**日期**: 2026-03-05

## 1. 项目概述

基于 ESP32 的 BLE 遥控机器人，支持通过蓝牙远程控制两个 PWM 舵机，具有平滑移动和预设动作功能。后续扩展 UART 与其他 MCU 通信。

## 2. 功能需求

- 多个舵机角度控制（2个舵机）
- 速度/平滑移动
- 预设动作/序列

## 3. 系统架构

### 分层架构

```
┌─────────────────────────────────────┐
│           应用层 (app_main)          │
├─────────────────────────────────────┤
│         命令解析层 (command)         │  ← 统一解析 BLE / UART 指令
├─────────────────────────────────────┤
│         动作管理层 (action)         │  ← 管理预设动作序列
├─────────────────────────────────────┤
│         舵机控制层 (servo)          │  ← PWM 控制舵机
├─────────────────────────────────────┤
│         BLE 通信层 (ble)            │  ← 蓝牙 GATT 服务
├─────────────────────────────────────┤
│         UART 通信层 (uart)          │  ← 与其他 MCU 通信 (后续扩展)
└─────────────────────────────────────┘
```

### 设计原则

- **命令解析层统一**：BLE 和 UART 都走同一个解析器，新增通信方式只需添加适配层
- **预留扩展接口**：UART 层暂不实现，但接口先定义好

## 5. 模块设计

### 5.1 命令解析层 (command)

负责解析来自 BLE 和 UART 的命令。

**命令格式**（示例）：
```
SET_SERVO_1:90,TIME:500   # 设置舵机1到90度，耗时500ms
SET_SERVO_2:45,TIME:300   # 设置舵机2到45度，耗时300ms
PLAY_ACTION:1              # 播放预设动作1
```

### 5.2 舵机控制层 (servo)

- 使用 MCPWM 或 LEDC 外设产生 PWM 信号
- 支持角度设置和移动时间参数
- 平滑移动实现：定时器插值

### 5.3 动作管理层 (action)

- 预设动作序列存储
- 动作录制/播放功能
- 动作队列管理

### 5.4 BLE 通信层 (ble)

- GATT 服务定义
- 特征值读写
- Notify 推送状态

### 5.5 UART 通信层 (uart)

**暂不实现**，接口预留：
- `uart_init()`
- `uart_send()`
- `uart_register_callback()`

## 6. GATT 服务定义

| UUID | 名称 | 描述 |
|------|------|------|
| 0x00FF | Service UUID | 主服务 |
| 0xFF01 | SERVO_CTRL | 舵机控制特征（读写） |
| 0xFF02 | ACTION_CTRL | 动作控制特征（读写） |
| 0xFF03 | STATUS | 状态推送特征（Notify） |

## 7. 后续扩展

### UART 通信

- 协议格式与 BLE 命令格式统一
- 支持主/从模式
- 波特率：115200（默认）

### 舵机数量扩展

- 架构支持多舵机
- 只需在 servo 层添加通道配置
